A Vim User's Exploration of Lua

Developing a Lua project on Windows is not as simple and straightforward as it may seem. This blog post hopes to address the gap in guidance by providing an opinionated approach to organizing Lua development on a Windows workstation.

First, we will install Lua and Luarocks using the MinGW64 subsystem (at the time of writing).

   pacman -S mingw-w64-x86_64-lua 5.4.6-2 

This will install the Lua interpreter (lua.exe) which allows you to run Lua scripts and develop applications using this versatile language.

   pacman -S mingw-w64-x86_64-lua-luarocks 3.9.2-1 

It will install Luarocks which serves as a package manager for Lua modules, enabling developers to easily install, manage, and share Lua libraries.

By including the MinGW64 path in the PowerShell PATH environment variable, the Lua interpreter (lua.exe) will be accessible from any PowerShell terminal. However, Luarocks is a bash script and must be run within the MinGW64 terminal, as it also requires a C compiler to function properly (in this configuration mingw-w64-x86_64-gcc 13.2.0-6 mingw-w64-x86_64-toolchain).

Vim9 Lua Interface

While Vim has a built-in Lua interface (see :h lua) when compiled with the +lua/dyn option, it is not well-suited for developing full-scale projects. The primary limitation is that it's really difficult to install modules using Luarocks, which restricts the use of external libraries, often essential for complex applications.

To check if your Vim supports Lua, you can run the following command:

   :echo has('lua')

Or as shown in the video below, run a Lua buffer with the command:

   :luafile %

Lua LSP

I have extensively discussed the Vim LSP plugin in many of my previous blog posts. For Lua development, there is the Lua Language Server available at LuaLS, which can be easily integrated into Vim LSP. To set it up, simply download the Win32 ZIP file from the releases page and unzip it into a directory of your choice.

Lua Debugger

As demonstrated in the video above, it is possible to use a debugger. debugger.lua is a simple, single-file, pure Lua debugger that is easy to integrate into Vim. To set it up, download the debugger.lua file from the repository https://github.com/slembcke/debugger.lua and include it in your project’s root folder.

Rocks

As an example, we will use a simple Lua script that sends a GET request to a remote server, which will return a random description of a common Vim keybinding in a table format. To accomplish this, we need to install an additional Lua module (commonly referred to as a 'rock') by using the command:

   luarocks install luasec

This module will enable us to handle HTTPS requests in our script.

Vim9lua

Finally, we have created a Vim9Script plugin that allows us to conveniently run and debug the following script in Vim.

local https = require("ssl.https")
local dbg = require("debugger")


function get_vimtips(url)
    local response, code, headers, status = https.request(url)
    dbg()
    if code == 200 then
        print(response)
    else
        print("Error: " .. status)
    end
end
get_vimtips("https://tessarinseve.pythonanywhere.com/vimtips")

The easiest way to install this plugin is by using the Vim-Plug plugin manager. Simply add the following line to your .vimrc file:

   Plug 'sevehub/vim9lua'

After adding the line, run :PlugInstall to install the plugin. Additionally, set the variable g:lualsp_path to the path of the LSP that you previously downloaded. This setup will provide you with two commands: :RunLua and :DebugLua, along with LSP integration.
