Crafting a Kanban Board with p5.js in JupyterLite

In the quest for better organization and productivity, the Kanban system has emerged as a powerful tool for managing tasks and projects. By providing a visual representation of work items, Kanban boards help users prioritize tasks, track progress, and enhance collaboration. In this blog post, we will guide you through the process of utilizing a Kanban board within the JupyterLite environment. This user-friendly approach allows you to seamlessly manage your tasks while enjoying the interactive features of JupyterLite.

As illustrated in the figure below, let's first define the four key steps of our Kanban workflow, utilizing the 2x2 layout of our index card notebook:

  1. Design Backlog: This card contains tasks and features extracted from product design that are yet to be worked on. It serves as a repository for all ideas and tasks that need to be prioritized.
  2. In Progress: This card represents tasks that are currently being worked on. It helps me focus on what’s actively being developed, ensuring that I stay on track and maintain momentum.
  3. Completed (with Issues): This card indicates tasks that have been completed but may have unresolved issues or challenges that need to be addressed. It allows me to acknowledge any obstacles while celebrating progress.
  4. Done (Final Observations): This final card signifies tasks that are fully completed, ensuring that any outstanding issues are acknowledged along with final observations. It’s a great way to wrap up tasks and reflect on what can be improved in future projects.

Using this index card system keeps my workflow organized and visually clear, making it easy to understand the status of each task on the Kanban board.

p5.js

p5.js is a powerful JavaScript library designed for creative coding, making it easy to create graphics and interactive content in a web environment. JupyterLite offers a kernel specifically for creating p5 notebooks, allowing users to seamlessly integrate p5.js into their projects. The Kanban board described here was created using the p5.js kernel in JupyterLite.

In p5.js, there are two essential functions: setup() and draw(). The setup() function is called once at the beginning of the program, where you can initialize variables, set the canvas size, and configure settings. In contrast, the draw() function continuously executes the lines of code contained within it, allowing for dynamic and interactive visuals that can change over time.

Here's the example showcased in our video, in case you want to follow along or explore the concepts further.

    let x = 0; // Starting position of the circle

    function setup() {
      createCanvas(400, 200); // Set the canvas size
    }

    function draw() {
      background(220); // Clear the background
      ellipse(x, height / 2, 50, 50); // Draw the circle
      x += 2; // Move the circle to the right
    }

FFMPEG

Finally, the introduction of the video features a segment where an image is superimposed onto an existing video. This technique works exceptionally well using the command-line program FFmpeg, which prompted me to include a note in this blog post. FFmpeg was installed via MSYS2/MinGW64 using the command

   pacman -S mingw-w64-x86_64-ffmpeg

and the JupyterLite logo superimposed using the following command, with x and y the position of the logo in pixels:

   ffmpeg -i input_video.mp4 -i overlay_image.png -filter_complex "overlay=x:y" -codec:a copy output_video.mp4

I found it to be not only enjoyable to use but also remarkably fast and stable at the time of writing.
